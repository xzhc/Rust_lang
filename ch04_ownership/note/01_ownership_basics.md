# 所有权基础

> 📖 对应书中章节：Chapter 4.1 - What Is Ownership?

## 🎯 什么是所有权？

**所有权（Ownership）** 是 Rust 最独特的特性，它让 Rust 能够在不使用垃圾回收器（GC）的情况下保证内存安全。

### 三种内存管理方式对比

| 方式 | 语言 | 特点 |
|------|------|------|
| 垃圾回收 (GC) | Java, Python, Go | 自动回收，但运行时开销 |
| 手动管理 | C, C++ | 灵活但容易出错（内存泄漏、悬垂指针） |
| **所有权系统** | **Rust** | 编译时检查，零运行时开销 |

---

## 🏗️ 栈（Stack）与堆（Heap）

理解所有权之前，先要理解栈和堆的区别。

### 栈的特点

- **后进先出（LIFO）**：像叠盘子一样
- **速度快**：不需要搜索位置，永远在栈顶操作
- **大小固定**：数据必须在编译时确定大小
- **自动管理**：函数结束时自动弹出

```rust
// 栈上的数据 - 固定大小
let x = 5;          // 整数，固定大小
let y = true;       // 布尔值，固定大小
let z = 'a';        // 字符，固定大小
```

### 堆的特点

- **动态分配**：大小可以在运行时改变
- **速度较慢**：需要找空间、做记录
- **通过指针访问**：指针本身在栈上，数据在堆上

```rust
// 堆上的数据 - 动态大小
let s = String::from("hello");  // String 在堆上
let v = vec![1, 2, 3];          // Vec 在堆上
```

### 类比理解

想象你去餐厅：

| 场景 | 对应概念 |
|------|----------|
| 餐厅是内存 | 内存空间 |
| 栈像一个自助餐台 | 拿了就走，简单快速 |
| 堆像点餐入座 | 服务员找位子、记录、带路 |
| 指针是桌号 | 通过桌号找到你的位置 |

> 💡 **为什么要关心栈和堆？**
> 所有权的主要目的是管理**堆上的数据**。栈上的数据会自动复制，但堆上的数据需要明确的所有者来负责清理。

---

## 📜 所有权三大规则

请务必记住这三条规则：

```text
1. Rust 中的每个值都有一个所有者（owner）
2. 同一时刻只能有一个所有者
3. 当所有者离开作用域，值将被丢弃（drop）
```

---

## 📍 变量作用域

**作用域（Scope）** 是变量有效的范围。

```rust
{                      // s 在这里无效，尚未声明
    let s = "hello";   // 从此刻起，s 有效

    // 使用 s 做一些事情
}                      // 作用域结束，s 不再有效
```

### 作用域的关键时间点

1. **进入作用域**：变量声明时变得有效
2. **离开作用域**：变量被自动清理

## 🔤 String 类型

为了演示所有权，我们需要一个存储在堆上的数据类型 —— `String`。

### 字符串字面值 vs String

| 类型 | 可变性 | 存储位置 | 使用场景 |
|------|--------|----------|----------|
| `"hello"` (字面值) | 不可变 | 编译时嵌入二进制 | 固定文本 |
| `String::from("hello")` | 可变 | 堆上 | 运行时动态文本 |

```rust
fn main() {
    // 字符串字面值 - 不可变，编译时已知
    let literal = "hello";

    // String 类型 - 可变，堆上分配
    let mut s = String::from("hello");
    s.push_str(", world!");  // 可以追加内容

    println!("{}", s);  // 输出: hello, world!
}
```

### String 的内存布局

一个 `String` 由三部分组成（存储在栈上）：

```text
┌─────────────────────┐
│  ptr (指针)         │ ──→ 堆上的数据 "hello"
│  len (长度) = 5     │
│  capacity (容量) = 5│
└─────────────────────┘
        栈上
```

## 🧹 内存和分配

### 分配内存

```rust
let s = String::from("hello");  // 在堆上分配内存
```

当我们调用 `String::from` 时，它的实现会：
1. 请求所需的内存
2. 返回一个 `String`

### 释放内存

在拥有 GC 的语言中，GC 会自动回收。在 C/C++ 中，需要手动 `free`/`delete`。

**Rust 的方式：自动释放**

```rust
{
    let s = String::from("hello");  // s 有效

    // 使用 s
}  // ← 作用域结束，Rust 自动调用 drop 函数，释放内存！
```

> 💡 **RAII 模式**
> 这种在作用域结束时自动释放资源的模式叫做 **Resource Acquisition Is Initialization (RAII)**，如果你熟悉 C++，应该不陌生。

---

## 📝 小结

1. **所有权是 Rust 内存管理的核心**，无需 GC 也能保证安全
2. **栈存储固定大小数据，堆存储动态数据**
3. **三个所有权规则**：
   - 每个值有且仅有一个所有者
   - 所有者离开作用域时值被释放
4. **String 类型**演示了堆数据的所有权行为
