# 移动、克隆与复制

> 📖 对应书中章节：Chapter 4.1 - Variables and Data Interacting with Move/Clone/Copy

## 🔄 变量与数据的交互

当多个变量与同一数据交互时，Rust 的行为与其他语言不同。让我们深入理解。

---

## 📦 移动（Move）

### 栈数据的简单复制

对于简单类型（如整数），赋值就是复制：

```rust
fn main() {
    let x = 5;
    let y = x;  // 复制 x 的值给 y

    println!("x = {}, y = {}", x, y);  // 两个都能用！
    // 输出: x = 5, y = 5
}
```

### 堆数据的移动

但是 `String` 类型不同！

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // 发生了什么？

    // println!("{}", s1);  // ❌ 编译错误！s1 已经无效
    println!("{}", s2);     // ✅ 只有 s2 有效
}
```

### 为什么会这样？

让我们看看内存中发生了什么：

**赋值前（s1）：**
```text
栈                          堆
┌─────────────────┐        ┌───────────────────┐
│ s1              │        │ index │ value     │
│  ptr ────────────────→   │   0   │ 'h'       │
│  len  = 5       │        │   1   │ 'e'       │
│  cap  = 5       │        │   2   │ 'l'       │
└─────────────────┘        │   3   │ 'l'       │
                           │   4   │ 'o'       │
                           └───────────────────┘
```

**赋值后（let s2 = s1）：**
```text
栈                          堆
┌─────────────────┐        ┌───────────────────┐
│ s1 (无效!)      │        │ index │ value     │
│  ptr ──┐        │        │   0   │ 'h'       │
│  len   │        │   ┌──→ │   1   │ 'e'       │
│  cap   │        │   │    │   2   │ 'l'       │
└────────│────────┘   │    │   3   │ 'l'       │
┌────────│────────┐   │    │   4   │ 'o'       │
│ s2     │        │   │    └───────────────────┘
│  ptr ──┘        │───┘
│  len  = 5       │
│  cap  = 5       │
└─────────────────┘
```

### 为什么不是复制堆数据？

如果复制堆数据：
- 性能差：大数据复制很慢
- 内存占用翻倍

### 为什么不是两个指针都有效？

如果两个指针都有效：
- 当 s1 和 s2 都离开作用域时，会**两次释放同一内存**
- 这叫 **double free** 错误，会导致内存损坏！

### Rust 的解决方案：移动

```rust
let s1 = String::from("hello");
let s2 = s1;  // s1 被"移动"到 s2，s1 不再有效
```

> 💡 **移动 vs 浅拷贝**
> 移动类似于浅拷贝（只复制指针），但**同时使原变量无效**。这避免了 double free 问题。

## 🔄 重新赋值也会 drop

当给变量赋予新值时，旧值会被立即释放：

```rust
fn main() {
    let mut s = String::from("hello");
    s = String::from("ahoy");  // "hello" 立即被 drop！

    println!("{}, world!", s);  // 输出: ahoy, world!
}
```

内存变化：
```text
之前: s → 堆上的 "hello"
之后: s → 堆上的 "ahoy"（"hello" 已被释放）
```

## 📋 克隆（Clone）

如果你**真的需要**深拷贝堆数据，使用 `clone()` 方法：

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone();  // 深拷贝！

    println!("s1 = {}, s2 = {}", s1, s2);  // 两个都有效！
}
```

内存变化：
```text
栈                          堆
┌─────────────────┐        ┌───────────────────┐
│ s1              │        │ 'h' 'e' 'l' 'l' 'o'│
│  ptr ────────────────→   └───────────────────┘
│  len  = 5       │
│  cap  = 5       │        ┌───────────────────┐
└─────────────────┘        │ 'h' 'e' 'l' 'l' 'o'│
┌─────────────────┐        └───────────────────┘
│ s2              │              ↑
│  ptr ─────────────────────────┘
│  len  = 5       │
│  cap  = 5       │
└─────────────────┘
```

> ⚠️ **注意**
> `clone()` 可能很昂贵（时间和内存）。看到 `clone` 就知道这里发生了深拷贝。

## 📋 复制（Copy）trait

为什么整数可以直接复制而 `String` 不行？

### Copy trait

某些类型实现了 `Copy` trait，这些类型在赋值时会自动复制，原变量仍然有效。

```rust
fn main() {
    let x = 5;
    let y = x;

    println!("x = {}, y = {}", x, y);  // 都能用！
}
```

### 哪些类型实现了 Copy？

| 类型 | 例子 |
|------|------|
| 所有整数类型 | `u8`, `u32`, `i64`, `isize` 等 |
| 浮点类型 | `f32`, `f64` |
| 布尔类型 | `bool` |
| 字符类型 | `char` |
| **只包含 Copy 类型的元组** | `(i32, i32)` ✅ 但 `(i32, String)` ❌ |

### Copy 与 Drop 互斥

如果一个类型或其任何部分实现了 `Drop` trait，就不能再实现 `Copy`。

```rust
// ❌ 不可能！String 实现了 Drop，不能实现 Copy
// 如果允许，double free 问题就会发生
```

---

## 📊 速查表

| 操作 | 栈数据 (Copy) | 堆数据 (非 Copy) |
|------|--------------|-----------------|
| 赋值 `let y = x` | 复制值，两者都有效 | 移动，原变量无效 |
| 克隆 `y = x.clone()` | 不需要 | 深拷贝，两者都有效 |
| 离开作用域 | 自动弹出 | 调用 drop 释放 |

---

## 🎯 判断是 Move 还是 Copy

问自己两个问题：

1. **这个类型是否实现了 Copy trait？**
   - 是 → 赋值时复制，原变量仍有效
   - 否 → 赋值时移动，原变量无效

2. **我需要两个变量都有有效数据吗？**
   - 是 → 使用 `.clone()`
   - 否 → 让所有权移动即可

## 📝 代码示例

```rust
fn main() {
    // ===== Copy 类型 =====
    let a = 5;
    let b = a;
    println!("a = {}, b = {}", a, b);  // 都能用

    // ===== Move 类型 =====
    let s1 = String::from("hello");
    let s2 = s1;
    // println!("{}", s1);  // ❌ 错误：s1 已移动
    println!("{}", s2);      // ✅ 正常

    // ===== Clone =====
    let s3 = String::from("world");
    let s4 = s3.clone();
    println!("s3 = {}, s4 = {}", s3, s4);  // 都能用

    // ===== 元组的 Copy =====
    let t1 = (1, 2);           // 两个 i32，Copy
    let t2 = t1;
    println!("{:?}, {:?}", t1, t2);  // 都能用

    let t3 = (1, String::from("hi"));  // 包含 String，不 Copy
    let t4 = t3;
    // println!("{:?}", t3);  // ❌ 错误：t3 已移动
    println!("{:?}", t4);      // ✅ 正常
}
```
