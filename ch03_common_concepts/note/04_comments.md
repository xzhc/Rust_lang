# 注释

> 📖 对应书中章节：Chapter 3.4 - Comments

注释是用来解释代码的文本，编译器会忽略它们。

## 📝 单行注释

Rust 使用 `//` 开始单行注释，注释持续到行尾：

```rust
fn main() {
    // 这是一个单行注释
    println!("Hello, world!");  // 注释也可以放在代码后面
}
```

### 多行注释

对于跨多行的注释，每行都需要 `//`：

```rust
fn main() {
    // 我们在这里做一些复杂的事情，
    // 复杂到需要多行注释来解释！
    // 希望这个注释能解释清楚。
    let x = 5;
}
```

> 💡 **惯例**：在 Rust 中，多行注释通常使用多个 `//`，而不是 `/* */`。

## 📍 注释的位置

### 行尾注释

```rust
fn main() {
    let lucky_number = 7;  // 我今天很幸运
}
```

### 独立行注释

```rust
fn main() {
    // 我今天很幸运
    let lucky_number = 7;
}
```

> 💡 **惯例**：更常见的是将注释放在代码上方，这样更易读。

## 📚 文档注释

Rust 还有**文档注释**，用于生成文档（将在第14章详细讨论）：

```rust
/// 这是一个文档注释
/// 它会生成 HTML 文档
fn add(a: i32, b: i32) -> i32 {
    a + b
}

//! 这是模块级别的文档注释
//! 用于描述整个模块或 crate
```

| 类型 | 符号 | 用途 |
|------|------|------|
| 普通注释 | `//` | 代码内解释 |
| 外部文档注释 | `///` | 函数、结构体等的文档 |
| 内部文档注释 | `//!` | 模块或 crate 的文档 |

## 🎯 最佳实践

```rust
fn main() {
    // 好的注释：解释"为什么"
    // 使用 saturating_add 防止溢出导致的安全问题
    let result = a.saturating_add(b);

    // 坏的注释：解释"是什么"（代码本身已经说明了）
    // 把 5 赋值给 x
    let x = 5;  // ← 这种注释是多余的
}
```

### 注释原则

1. **解释为什么，而不是做什么**：代码本身应该清晰表达"做什么"
2. **保持更新**：修改代码时记得更新相关注释
3. **不要过度注释**：好的代码是自解释的
4. **使用文档注释**：公共 API 应该有文档注释

## 📝 速查表

```rust
// 单行注释

// 多行注释
// 第二行
// 第三行

/// 文档注释（用于函数、结构体等）
fn example() {}

//! 模块级文档注释
```

## 🎯 练习建议

1. 为你之前写的函数添加注释
2. 尝试使用 `cargo doc --open` 查看文档注释的效果
3. 实践区分"好的注释"和"坏的注释"
