# 变量与可变性

> 📖 对应书中章节：Chapter 3.1 - Variables and Mutability

## 🔤 变量默认不可变

在 Rust 中，变量**默认是不可变的**（immutable）。这是 Rust 的一个重要设计决策，旨在帮助你编写更安全、更易于并发处理的代码。

```rust
fn main() {
    let x = 5;
    println!("x 的值是: {x}");
    x = 6;  // ❌ 编译错误！不能给不可变变量二次赋值
    println!("x 的值是: {x}");
}
```

编译这段代码会得到错误：

```
error[E0384]: cannot assign twice to immutable variable `x`
 --> src/main.rs:4:5
  |
2 |     let x = 5;
  |         - 第一次赋值
3 |     println!("x 的值是: {x}");
4 |     x = 6;
  |     ^^^^^ 不能给不可变变量二次赋值
```

> 💡 **编译器是你的朋友**：Rust 的编译错误非常友好，它不仅告诉你哪里错了，还会给出修复建议。

## 🔄 使用 `mut` 声明可变变量

当你确实需要改变变量的值时，可以使用 `mut` 关键字：

```rust
fn main() {
    let mut x = 5;  // 加上 mut，x 变成可变变量
    println!("x 的值是: {x}");
    x = 6;          // ✅ 现在可以修改了
    println!("x 的值是: {x}");
}
```

输出：
```
x 的值是: 5
x 的值是: 6
```

### 为什么要默认不可变？

1. **减少 bug**：如果代码的某部分假设一个值不会改变，而另一部分却改变了它，可能会导致难以追踪的 bug
2. **并发安全**：不可变数据天生是线程安全的
3. **代码推理**：不可变变量更容易理解和推理

> ⚠ **注意**：选择可变还是不可变，取决于你的具体需求。Rust 让你显式地表达意图。

## 📦 常量（Constants）

常量类似于不可变变量，但有一些重要区别：

```rust
const THREE_HOURS_IN_SECONDS: u32 = 60 * 60 * 3;
```

### 常量 vs 变量

| 特性 | 常量 (`const`) | 不可变变量 (`let`) |
|------|---------------|-------------------|
| 关键字 | `const` | `let` |
| 类型标注 | **必须** | 可选（可推断） |
| `mut` | 不允许 | 可以配合使用 |
| 赋值时机 | 编译时 | 运行时 |
| 作用域 | 可在任何作用域声明 | 受限于所在作用域 |

### 常量的命名约定

- 使用**全大写字母**
- 单词之间用**下划线**分隔

```rust
const MAX_POINTS: u32 = 100_000;  // 下划线提高可读性
const SPEED_OF_LIGHT: f64 = 299_792_458.0;
```

> 💡 常量在整个程序运行期间都有效，非常适合存储程序中多处使用的硬编码值。

## 👥 变量遮蔽（Shadowing）

Rust 允许你用同一个名字声明多个变量，后面的变量会**遮蔽**（shadow）前面的：

```rust
fn main() {
    let x = 5;           // 第一个 x

    let x = x + 1;       // 第二个 x 遮蔽了第一个，值为 6

    {
        let x = x * 2;   // 第三个 x（内部作用域），值为 12
        println!("内部作用域中 x 的值是: {x}");
    }

    println!("x 的值是: {x}");  // 回到第二个 x，值为 6
}
```

输出：
```
内部作用域中 x 的值是: 12
x 的值是: 6
```

### Shadowing vs `mut`

Shadowing 和 `mut` 的关键区别：

```rust
// ✅ 使用 shadowing：可以改变类型
let spaces = "   ";        // &str 类型
let spaces = spaces.len(); // usize 类型，没问题！

// ❌ 使用 mut：不能改变类型
let mut spaces = "   ";
spaces = spaces.len();     // 编译错误！类型不匹配
```

| 特性 | Shadowing | `mut` |
|------|-----------|-------|
| 重复声明 | 需要 `let` | 不需要 |
| 改变类型 | ✅ 可以 | ❌ 不行 |
| 编译时检查 | 更严格 | 相对宽松 |

> 💡 **实际应用**：当你需要对一个值进行多次转换，但每次转换后不需要保留中间值时，shadowing 非常有用。

## 📝 速查表

```rust
// 不可变变量（默认）
let x = 5;

// 可变变量
let mut y = 10;
y = 20;  // OK

// 常量
const MAX_VALUE: u32 = 100;

// 变量遮蔽
let z = 5;
let z = z + 1;  // 新的 z，值为 6
let z = "hello"; // 甚至可以改变类型！
```

## 🎯 练习建议

1. 尝试创建一个不可变变量并修改它，观察编译器错误
2. 使用 `mut` 修复上面的错误
3. 创建一个常量并尝试用 `mut` 修饰它（会失败）
4. 练习 shadowing：将一个字符串变量遮蔽为它的长度
